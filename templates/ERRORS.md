# ERRORS.md — {{PROJECT_NAME}}

> Failure log. Check before suggesting an approach to a problem that resembles past work. Log only when an approach took more than two attempts to get right — keep the signal-to-noise high.

## Format

```
## YYYY-MM-DD: <one-line description of the problem>
- **What didn't work:** ...
- **Why it failed:** ...
- **What worked instead:** ...
- **Note for next time:** ...
- **Files / areas affected:** ...
```

## Threshold rule

Only log entries when an approach **took more than two attempts to work**. One-shot fixes belong in the implementation, not here. Flooding ERRORS.md with trivial misfires defeats its purpose — the value comes from catching repeated mistakes, and that only works if every entry is real signal.
